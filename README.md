# 多人 AI 生图网站使用说明

这份文档面向团队成员，帮助大家快速上手使用网站。

## 1. 功能简介

网站支持多人同时使用，包含以下能力：

- 访问者身份登记（用于记录是谁生成的图）
- 文本生图
- 参考图生图（按模型能力决定是否支持）
- 多模型切换
- 生成记录查看
- 管理员后台配置模型与 API

## 2. 普通用户怎么用

1. 打开网站后，先输入你的姓名或工号作为访问者身份。
2. 在主界面填写 `Prompt`。
3. 如果需要，上传参考图（支持拖拽/多选）。
4. 选择模型、图片比例、图片张数。
5. 点击“生成图片”。
6. 生成完成后，在输出区查看、预览、下载图片。
7. 右侧“本机记录”可以查看当前这台电脑上的历史生成记录。

## 3. 管理员怎么用

默认管理员账号：

- 用户名：`admin`
- 密码：`1596357`

首次使用建议立即修改管理员密码。

管理员操作步骤：

1. 点击页面右上角齿轮按钮进入后台。
2. 登录管理员账号。
3. 在“API / 模型管理”中新增或编辑模型。
4. 填写字段：
   - 显示名称：前端下拉框看到的名字
   - 模型 ID：调用接口用的 model 字段
   - API 地址：供应商图片生成接口地址
   - API Key：供应商密钥
   - 尺寸：默认输出尺寸（例如 `1024x1024`、`2048x1152`）
   - 启用：是否在前端显示该模型
5. 保存后，模型会出现在生图界面的模型选择框里。

## 4. 不同模型的参考图说明

不同供应商接口能力不同，常见情况：

- 有的模型支持直接上传参考图做图生图
- 有的模型只支持文生图
- 有的模型要求走 `images/edits` 才能使用参考图

如果某个模型报参数错误（如 `Unknown parameter`），通常是该模型接口不支持当前传参方式，需要在服务端做适配。

## 5. 生成记录说明

记录中会展示：

- 使用者
- 时间
- 生成模型
- 成功/失败状态
- 参考图按钮
- 输出结果按钮
- Prompt 与生成数量

说明：

- 输出结果如果是大体积 base64，系统会落盘到 `data/generated/`，记录里保存短链接，避免记录文件过大导致记录不可见。

## 6. 常见问题

### 6.1 出现 `fetch failed`

常见原因：

- API 地址不可达（网络/防火墙/DNS）
- API 服务超时或临时故障
- HTTPS 证书/TLS 问题
- 模型接口路径不对（例如应为 `/v1/images/...`）

排查建议：

1. 在后台确认模型的 `endpoint` 和 `apiKey`。
2. 在服务器上用 `curl` 最小请求测试该接口。
3. 查看服务日志（如 `pm2 logs ai-image-site`）定位具体上游错误。

### 6.2 生成成功但记录里看不到图

先确认：

- 服务进程是否有 `data/` 写入权限
- `data/generated/` 是否正常创建
- 记录里是否已有“输出结果”按钮

## 7. 本地运行

环境要求：

- Node.js `>= 18`

启动命令：

```bash
npm start
```

访问地址：

```text
http://127.0.0.1:3000
```
